# Differences Between VK and HID

**Windows VK codes (Virtual‑Key Codes):**
- Used by Windows OS.
- Each key has a **single hex value** (ex: `0x45` = E).
- Doesn’t distinguish between lowercase or symbols—Shift determines that separately.

**macOS HID Usage codes:**
- These come from USB HID (Human Interface Device) standards.
- macOS uses **scan‑code style** values for the physical key position.
- Also does **not** change based on shift (Shift just modifies output).

---

# Every symbol key is **one physical key** with **two characters**:

#### Windows (VK 0xBF)
- `/` → **no shift held**  
- `?` → **shift held**

**Example Usage:**

#### Windows (VK 0xBF)
- `/` → **no shift held**  
- `?` → **shift held**

**Example Usage:**

```lua
-- Press / (no shift)
keypress(0xBF)
keyrelease(0xBF)

-- Press ? (shift held)
keypress(0x10)  -- hold Shift (VK 0x10)
keypress(0xBF)  -- press /
keyrelease(0xBF)
keyrelease(0x10) -- release Shift
```

**Explanation:**  
1. Without Shift → `/`  
2. With Shift → hold `Shift` + `/` → `?`

---

#### macOS (HID 0x2C)
- `/` → **base output (no shift held)**  
- `?` → **shift-modified output (shift held)**  
- Shift key → `0x38`

**Example Usage:**

```lua
-- Press / (no shift)
keypress(0x2C)
keyrelease(0x2C)

-- Press ? (shift held)
keypress(0x38)  -- hold Shift
keypress(0x2C)  -- press /
keyrelease(0x2C)
keyrelease(0x38) -- release Shift
```

**Explanation:**  
1. Without Shift → `/`  
2. With Shift → hold Shift + `/` → `?`
